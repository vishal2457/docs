# Why

There are a lot of templates and boilerplates available, but most of them are overly bloated.
You/Your team spends a lot of time removing unnecessary code which lowers the productivity instead of increasing it.

Our aim with maximus is to build software with industry standards and fast. Utilize various devtools and empower teams to deliver features fast.
Maximus starter kits comes with dev tools pre-configured.

## Setting up

Setting up a project with maximus is really easy.

1. Select technologies.
2. Select modules you want in the project.
3. Hit download. Maximus will generate an opinionated starter kit with best practices for your project.
4. Start your project with instructions in README.

## Currently Supported

### Full stack app with following

- Nx/Turborepo
- Angular
- Express with (postgres / MYSQL / sqlite)
- [Typescript](https://www.typescriptlang.org/)
- [Tailwind CSS](https://tailwindcss.com/docs/installation)
- [Docker](https://www.docker.com/)
- Github actions/ Gitlab CI
- [Semantic Release for versioning](https://semantic-release.gitbook.io/semantic-release).
- Generate automatic change logs from commit messages.
- Eslint
- Prettier
